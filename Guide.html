INTRO 10 min

What You Need & How This Works

Before building, understand what the bot does and what you need to have ready.

This bot connects to your WhatsApp via QR scan (like WhatsApp Web), listens for incoming messages, and uses Groq AI — loaded with your personal writing style — to reply on your behalf. Every reply passes through a human-voice filter so it doesn't sound AI-generated.

⚡
What you need before starting A Windows PC · Google Chrome installed · A WhatsApp account on your phone · A GitHub account (free at github.com) · A Groq account (free at console.groq.com)

How the bot works — end to end:

Someone messages you → bot receives it via WhatsApp Web connection
Message is sent to Groq AI with your persona loaded as the system prompt
Groq generates a reply in your voice (3–8 second human delay)
Reply is sent back — the contact has no idea it's automated
Everything is logged to bot.log, sensitive messages flagged for you
⚠️
Important limits to know upfront The bot handles text only — it skips voice notes, images, stickers, and videos. It won't know your real schedule or personal commitments. Check bot.log daily for ⚠️ HUMAN REVIEW NEEDED flags.
PHASE 1 15 min

Foundation — Install Node.js & Dependencies

Get the tools your computer needs to run the bot.

1

Go to nodejs.org and download the LTS version (the left/recommended button). Run the installer, click Next through everything using default options. When it asks about "Install tools for Native Modules" — check that box.

2

Once installed, close and reopen your terminal. Verify Node is working:

terminalnode --version

You should see something like v20.x.x or v24.x.x

3

Open your project folder in the terminal. In File Explorer, navigate to your project folder, click the address bar, type cmd, and press Enter.

Your terminal should show the full project path — not C:\Windows\System32

4

Install all bot dependencies:

terminalnpm install

Takes 1–3 minutes. You'll see packages being added. The important ending line is: found 0 vulnerabilities. Yellow warnings are normal — ignore them.

🔧
Troubleshooting If npm install gives a MODULE_NOT_FOUND error — you're in the wrong folder. Use the File Explorer method above to open the terminal directly inside the project folder.
✓ Phase 1 complete
PHASE 2 5 min

Get Your Free Groq API Key

Connect the bot to Groq AI — completely free, no credit card needed.

💡
Why Groq and not Google Gemini or Anthropic? Groq offers a genuinely free API tier that works in Nigeria. Gemini's free tier has a limit of 0 in certain regions. Anthropic requires payment upfront. Groq's Llama model is fast, capable, and costs nothing for personal use.
1

Go to console.groq.com and sign up with your Google account.

2

Click "API Keys" in the left sidebar → "Create API key". Copy the key — it starts with gsk_...

3

In your project folder, copy .env.example and rename the copy to .env (just .env — no "example"). Open it in Notepad and fill in:

.env GROQ_API_KEY=gsk_your_key_here YOUR_NAME=line upon line GROQ_MODEL=llama-3.3-70b-versatile REPLY_IN_GROUPS=false

Replace line upon line with whatever name appears in your WhatsApp chat exports. You'll confirm this in Phase 3.

4

Verify the key loaded correctly:

terminal — run from project foldernode -e "require('dotenv').config(); const k = process.env.GROQ_API_KEY; console.log(k ? 'Key loaded: ' + k.slice(0,8) + '...' : 'Key not found')"

You should see: Key loaded: gsk_XXXX...

🔧
MODULE_NOT_FOUND on the verify command? Your terminal is in the wrong folder — the dotenv package doesn't exist there. Navigate back to the project folder first using: cd "C:\Users\DELL\OneDrive\Documents\Claude\Projects\My whatsapp automation"
✓ Phase 2 complete
PHASE 3 20 min

Teach the Bot Your Voice

This is the most important phase. The bot is only as good as the persona you feed it.

We export your real WhatsApp chats, extract only your messages, and send them to Groq AI which analyzes your tone, vocabulary, emoji use, sentence style, and slang — then writes a detailed persona prompt saved to persona.txt.

1

Export WhatsApp chats from your phone. For each chat you want to export:

Open WhatsApp → any chat → tap the 3-dot menu → More → Export chat → Without Media

Send the .zip file to yourself via email or Google Drive. Download it, unzip it — you'll find a .txt file inside. Aim for 3–5 different chats where you write a lot (friends, family, colleagues).

2

Create a folder called chats inside your project folder. Drop all your .txt files there.

3

Run the parser to extract your messages:

terminalnode parse-chats.js

You should see: ✅ Parsed X messages from you. — the more messages the better. 200+ is good; 1000+ is excellent.

4

Generate your persona:

terminalnode generate-persona.js

Takes ~15 seconds. Groq analyzes 200 of your messages and writes a detailed persona file saved to persona.txt. Open it in Notepad and read it — add anything it missed.

5

Review and improve persona.txt. Open it in Notepad and manually add any habits the AI missed. For example:

- When someone asks for money, decline warmly but firmly
- Never discuss politics or gossip about others
- With close friends, skip formal greetings entirely
- Always respond to spiritual questions with encouragement
🔧
Could not find any messages from "your name"? Open one of your .txt chat files and look at the left side of a message you sent. That exact name (e.g. "line upon line") is what you need to set in YOUR_NAME in your .env file. The parser is case-sensitive and whitespace-sensitive — trim any trailing spaces.
🔧
Senders found but still 0 messages? Your chat export uses 24-hour time (e.g. 15:12) instead of 12-hour (3:12 PM). The parser handles both formats — make sure you're using the latest version of parse-chats.js from this session.
→ Go further Re-run generate-persona.js any time you want to refresh the persona. Add more .txt chat files to the /chats folder first for a richer result. The more variety (different people, different contexts), the better it captures your full range.
✓ Phase 3 complete
PHASE 4 5 min

Go Live — Scan QR & Test

Connect the bot to your WhatsApp and confirm it replies in your voice.

1

Start the bot:

terminalnode index.js

A QR code will appear in the terminal within 10–15 seconds.

2

On your phone: open WhatsApp → Settings → Linked Devices → Link a Device. Scan the QR code. You have about 60 seconds before it expires — if it does, a new one appears automatically.

3

Wait for the terminal to show:

🤖 Bot is ready! It will now reply to incoming messages.
4

Test it. Have a friend or second phone send your number a message. Watch the terminal — you'll see it log the incoming message and the reply. Check if the reply sounds like you.

5

If the reply doesn't sound right, stop the bot (Ctrl+C), edit persona.txt, and restart with node index.js.

🔧
Could not find Chrome error? The bot looks for Chrome in your Program Files. If you have Chrome installed in a non-default location, add this line to your .env file: CHROME_PATH=C:\path\to\chrome.exe
🔧
Puppeteer trying to download Chrome? Do NOT let it download — it can take hours and often fails. The bot is configured to use your installed Chrome automatically. If it still tries to download, cancel it (Ctrl+C) and check that your Chrome is installed at the standard path.
✓ Phase 4 complete
PHASE 5 30 min

Deploy 24/7 to Railway

Push your code to GitHub and deploy to Railway so the bot runs even when your laptop is off.

1

Install Git from git-scm.com/download/win. Run the installer with all default options. Close and reopen your terminal after installation.

2

Set your Git identity (do this once):

terminal git config --global user.email "your@email.com"
git config --global user.name "Your Name"
3

Create a private GitHub repo at github.com/new. Name it whatsapp-ai-bot. Set it to Private. Do NOT initialize with README or .gitignore.

4

Push your code to GitHub:

terminal — run each line separately git init
git add .
git commit -m "Initial bot setup"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/whatsapp-ai-bot.git
git push -u origin main

When asked for a password, use a GitHub Personal Access Token (not your password). Get one at github.com/settings/tokens/new — tick the repo scope, set No Expiration.

5

Upload your WhatsApp session so Railway can stay logged in:

terminal git add -f .wa-session
git commit -m "Add WhatsApp session"
git push
6

Deploy to Railway. Go to railway.app → sign in with GitHub → New Project → GitHub Repository → select whatsapp-ai-bot → Deploy Now.

7

Add your environment variables in Railway. Click your service → Variables tab. Add:

GROQ_API_KEY → your gsk_... key
GROQ_MODEL → llama-3.3-70b-versatile
YOUR_NAME → line upon line (or your name)
REPLY_IN_GROUPS → false
MIN_REPLY_DELAY_MS → 3000
MAX_REPLY_DELAY_MS → 8000
CONTEXT_WINDOW → 10

Save changes — Railway redeploys automatically.

8

Go to Deployments tab and wait for Deployment successful. Then click Deploy Logs — you should see:

WhatsApp authenticated ✅
🤖 Bot is ready! It will now reply to incoming messages.
🔧
Railway crashes with GROQ_API_KEY missing? The variables weren't saved before deployment. Go to Variables tab, confirm the key is there, save, and it will redeploy automatically.
🔧
QR code shows in Railway logs but can't scan it? Terminal QR codes in Railway logs are often too distorted to scan. The better approach: run the bot locally first (node index.js), scan the QR in your terminal, wait for "Bot is ready", then stop it (Ctrl+C). The session is now saved in .wa-session/. Push that to GitHub with git add -f .wa-session and Railway will use it automatically — no QR scan needed on the server.
→ Go further — keep it updated Any time you improve persona.txt or change the code, push updates to Railway with: git add . && git commit -m "update" && git push. Railway redeploys automatically within 1–2 minutes.
✓ Phase 5 complete — bot is live 24/7
